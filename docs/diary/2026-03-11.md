## Q&A Log

**Q: When we click on create space, if user doesn't enter name or select icon, just show a small popup to enter name.**
A: Added `createError` state in `SpaceSelector.tsx`. Validation triggers on "Create" click — shows inline error message if name is empty. Same added for edit flow with `editError` state.

---

**Q: Make username on spaces page bigger. New space + icon, put it in a corner.**
A: Username text size increased. Moved "+ New Space" button to a fixed bottom-right FAB (Floating Action Button) with a `+` icon.

---

**Q: When we click on edit spaces, the pencil and delete icon on each space looks plain. Add more style.**
A: Styled overlay buttons with gradients — gold gradient for pencil (edit), coral gradient for delete. Added hover scale animations.

---

**Q: Remove the label "Invite members by email..." — feels redundant.**
A: Removed the redundant label from the invite section in SpaceSelector.

---

**Q: In the invitation accept or reject, show the group icon. Right now it's default.**
A: Added `spaceIcon?: string` to `PendingInvite` type. Backend `/my-invites` now returns `coverIcon`. Frontend renders `SpaceIconRenderer` in the invite list instead of the default icon.

---

**Q: When we click on members button under each icon, make the popup a little smaller.**
A: Changed members modal max-width to `max-w-sm` (was `max-w-lg`).

---

**Q: If a user rejected a group invite and reloads the page, the notification still shows. Fix it.**
A: Backend `/my-invites` route now filters `status: 'pending'` only, so rejected invites are excluded from the response.

---

**Q: Since this is a React app without URLs, clicking browser back exits the application.**
A: Implemented browser History API. On space open, push `{ view: 'timeline', spaceId }` to `window.history`. On memory open, push `{ view: 'memory', memoryId }`. Added `popstate` listeners in `App.tsx` and `Timeline.tsx` to handle back navigation within the app using `skipNextPush` refs to avoid double-pushing.

---

**Q: Timeline page has too much space between center line and memory card. Bring them closer.**
A: Fixed `justify-start/end` → `justify-end/start` in `MemoryCard.tsx` to flip card alignment toward center line. Reduced `space-y-12` → `space-y-8` and `pb-40` → `pb-16` in `Timeline.tsx`.

---

**Q: Give 5 timeline design previews with our app theme.**
A: Created 5 HTML preview files exploring different timeline layouts. User rejected all — they didn't match the current theme and color palette well enough.

---

**Q: When a space/memory is deleted, are we deleting all photos?**
A: Backend `DELETE /spaces/:id` now: collects all photo URLs from memories + substories, deletes the space (cascade deletes memories via Prisma), responds immediately, then async-cleans Cloudinary. File: `MemoryWall-Claude backend/src/routes/spaces.ts`.

---

**Q: On delete, show a confirmation popup warning that memories and photos are permanently deleted.**
A: Added `deleteConfirmSpace` state in `SpaceSelector.tsx`. Clicking delete icon opens a modal with a permanent warning. Confirmed delete triggers `deleteSpace()`.

---

**Q: When a user has no spaces, show something on the space page.**
A: Added empty state with 3 icons (`friends-trio-0`, `reading-0`, `road-trip-0`), description text, and a CTA "+ New Space" button. Iterated styling multiple times based on feedback — final uses visible colored text and proper contrast.

---

**Q: Show random headings on the spaces page.**
A: Added `spacePageHeadings` and `spacePageSubheadings` arrays. Used `useState(() => ...)` initializer to pick random values once on mount.

---

**Q: The scroll of icons in create/edit space popup looks odd. Make it better.**
A: Redesigned icon picker — replaced vertical category list + inner scroll with category tabs. First tried horizontal scrolling tabs, then switched to wrapped pills (`flex-wrap`) based on feedback. Category short names used: Friends, Family, Trips, etc.

---

**Q: 2 "+ New Space" buttons are showing. Fix it.**
A: FAB only shown when `visibleSpaces.length > 0`. Center CTA button only shown in empty state. No overlap.

**Q: I want a mobile app for our project. How to proceed?**
A: Set up Capacitor:
- Installed `@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`, `@capacitor/android` + plugins
- Created `capacitor.config.ts` with splash screen (warm bg `#ffe8d6`), status bar, push notifications, camera
- Added `cap:ios`, `cap:android`, `cap:sync` scripts to `package.json`
- Built web app, added iOS and Android platforms
- `ios/` and `android/` folders created

---

**Q: In the account button (top right) and its content (edit, change password), update CSS to match theme.**
A: Restyled profile dropdown — colored icon circles, avatar header, warm theme colors. Reduced dropdown width to `w-60`.

---

**Q: Deployment failed — TS error on `selectedCategory` state.**
A: Root cause: `tsconfig.app.tsbuildinfo` was committed to git and the deployment platform used the stale cache (which had the old type error). Fixed by:
1. Adding `*.tsbuildinfo` to `.gitignore`
2. Running `git rm --cached tsconfig.app.tsbuildinfo`

---

**Q: Do I need to download Xcode to simulate iOS?**
A: Yes. Xcode (free, ~15GB) from Mac App Store. macOS only. Once installed: `npm run cap:ios` opens the project in Xcode → select simulator → hit ▶.
